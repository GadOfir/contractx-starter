## Gap entry template

```markdown
# CX-NNN — Short title

- **Status:** OPEN
- **Affects:** `lib/CONTRACTX.md`, `lib/handlers/CONTRACTX.md`
- **Discovered:** YYYY-MM-DD
- **Contradiction:** <one line>
- **Recommended resolution:** <one line, which file changes>
- **Owner:** <who fixes it>
```

## Lifecycle

1. **OPEN** — gap discovered, filed in `open/CX-NNN.md`
2. **SHIPPED** — fix committed, gap stays in `open/` until verified
3. **RESOLVED** — handoff verified, gap moved to `resolved/CX-NNN.md`
